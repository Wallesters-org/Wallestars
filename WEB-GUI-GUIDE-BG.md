## Бързо Стартиране

### Вариант 1: Най-лесният начин (с npm)

```bash
# Стъпка 1: Инсталирай зависимости (само първия път)
npm run setup

# Стъпка 2: Стартирай GUI платформата
npm start
```

### Вариант 2: Директно стартиране

```bash
# Стартирай веднага
node web-gui-server.js
```

### Стъпка 3: Отвори в браузъра

След стартиране отвори браузър и посети:
```
http://localhost:3000
```

**Готово!** 🎉 Виждаш красив GUI интерфейс на български език.

## Какво е Web GUI?

Web GUI Platform е **най-лесният начин** да използвате Wallestars Browser Automation.

### ✨ Предимства:

- ✅ **Без командна линия** - всичко е с бутони
- ✅ **Визуален интерфейс** - виждаш какво правиш
- ✅ **Без credentials** - работи и без GitHub парола
- ✅ **На български** - всичко е преведено
- ✅ **Лесно управление** - добавяй, премахвай, стартирай задачи
- ✅ **Real-time статус** - виждаш прогреса в реално време
- ✅ **Логове на живо** - проследяваш какво се случва

## Стъпка по Стъпка Инструкции

### 1️⃣ Стартиране на Сървъра

```bash
npm start
```

Ще видиш:
```
╔════════════════════════════════════════════════════════╗
║   🚀 Wallestars Browser Automation Platform 🚀        ║
╚════════════════════════════════════════════════════════╝

✓ Сървърът е стартиран на: http://0.0.0.0:3000

📖 Инструкции:
   1. Отворете браузър и посетете: http://localhost:3000
   2. Добавете задачи чрез интерфейса
   3. Credentials са опционални
   4. Натиснете "Старт" за изпълнение

⏹️  За спиране натиснете Ctrl+C
```

### 2️⃣ Отваряне на GUI Интерфейса

1. Отвори браузър (Chrome, Firefox, Safari, Edge - всички работят)
2. Посети: `http://localhost:3000`
3. Ще видиш красив, цветен интерфейс

### 3️⃣ Добавяне на Задачи

#### Метод 1: Ръчно въвеждане
1. В полето "Въведете задача" напиши задача
2. Натисни "➕ Добави" или Enter
3. Задачата се появява в списъка

#### Метод 2: Примерни задачи
1. Натисни бутона "📄 Примерни Задачи"
2. Автоматично се зареждат 4 примерни задачи

**Примери за задачи:**
- `Отвори GitHub и навигирай към Wallestars repository`
- `Прегледай отворените issues`
- `Провери последните commits`
- `Отвори Codespaces секцията`
- `Създай нов branch`

### 4️⃣ Опционални Настройки (Credentials)

💡 **Важно:** Credentials НЕ СА ЗАДЪЛЖИТЕЛНИ!

Ако искаш автоматично да влезеш в GitHub:
1. Въведи GitHub Email
2. Въведи GitHub Password

Ако оставиш празно:
- ✅ Платформата ще работи без влизане
- ✅ Браузърът ще се отвори на GitHub
- ✅ Можеш сам да влезеш ръчно ако е нужно

### 5️⃣ Стартиране на Изпълнението

1. Провери че имаш добавени задачи
2. Натисни големия зелен бутон "▶️ Старт"
3. Гледай как:
   - Статусът става "Активен"
   - Progress bar се пълни
   - Логовете показват какво се случва
   - Браузър се отваря автоматично

### 6️⃣ Наблюдаване на Прогреса

Докато задачите се изпълняват виждаш:

- **Статус Badge**: Показва дали е активен или неактивен
- **Progress Bar**: Процент на завършени задачи
- **Текуща Задача**: Коя задача се изпълнява в момента
- **Логове**: Всяко действие се записва в реално време

### 7️⃣ Спиране (ако е нужно)

1. Натисни червения бутон "⏹️ Стоп"
2. Изпълнението спира моментално
3. Браузърът се затваря

## Използване без Credentials

### Защо е възможно без credentials?

Платформата може да:
- Отваря GitHub в браузър
- Навигира до различни страници
- Показва информация
- Работи с публични repositories

### Кога СА нужни credentials?

Само ако искаш:
- Автоматично влизане в GitHub
- Достъп до частни repositories
- Създаване/редактиране на съдържание

### Как работи без credentials?

```
1. Стартираш платформата
2. Добавяш задачи
3. Натискаш Старт
4. Браузърът се отваря на github.com
5. Задачите се изпълняват с публичен достъп
6. Ако трябва - влизаш ръчно в браузъра
```

**Резултат:** Платформата работи перфектно! 🎉

## Често Задавани Въпроси

### ❓ Трябва ли да инсталирам нещо допълнително?

**Не, ако вече си пуснал `npm run setup`.**

Това инсталира:
- Node.js пакети (playwright и др.)
- Chromium браузър за автоматизация

### ❓ Работи ли на всяка операционна система?

**Да!**
- ✅ Windows
- ✅ macOS
- ✅ Linux
- ✅ GitHub Codespaces

### ❓ Трябва ли да въведа credentials?

**НЕ!** Credentials са 100% опционални.

Използвай credentials само ако:
- Искаш автоматично влизане
- Работиш с частни repositories
- Искаш да правиш промени в GitHub

### ❓ Мога ли да използвам от друго устройство?

**Да!** Промени адреса:

Вместо `localhost:3000` използвай IP адреса:
```
http://192.168.X.X:3000
```

Намери IP адреса:
- Windows: `ipconfig`
- Mac/Linux: `ifconfig` или `ip addr`

### ❓ Какво ако имам грешка при стартиране?

**Провери:**

1. Playwright браузъри инсталирани ли са?
   ```bash
   npm run install-browsers
   ```

2. Портът 3000 зает ли е?
   ```bash
   # Променете порта:
   PORT=8080 npm start
   ```

3. Node.js версия 16+ ли е?
   ```bash
   node --version
   ```

### ❓ Мога ли да променя порта?

**Да!**

```bash
# Опция 1: Environment variable
PORT=8080 npm start

# Опция 2: В кода
# Редактирай web-gui-server.js
# Промени: const PORT = process.env.PORT || 3000;
```

### ❓ Как да спра сървъра?

В терминала натисни: **Ctrl + C**

### ❓ Колко задачи мога да добавя?

Няма лимит! Добавяй колкото искаш.

### ❓ Задачите запазват ли се?

**Не** - при презареждане на страницата се изчистват.

Ако искаш да запазиш задачи:
1. Копирай ги в текстов файл
2. Или използвай `tasks.json` файла

### ❓ Мога ли да спра една задача по средата?

**Да!** Натисни бутона "⏹️ Стоп"

Това спира цялото изпълнение веднага.

### ❓ Виждам ли какво прави браузърът?

**Да!** Браузърът се отваря видимо (не headless).

Виждаш всяка стъпка в реално време:
- Отваряне на страници
- Навигация
- Кликване на бутони
- Всичко!
