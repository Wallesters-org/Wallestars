# Copilot Agent Instructions for Wallestars

This file provides custom instructions for GitHub Copilot coding agent when working on the Wallestars repository.

## Project Overview

Wallestars is a project in early development. The repository structure is minimal and should be kept simple and maintainable.

## Code Standards

### General Guidelines
- Write clear, self-documenting code
- Keep files and functions focused and concise
- Follow established patterns in the codebase
- Maintain consistency with existing code style

### Documentation
- Update the README.md when adding new features or changing functionality
- Include inline comments only when necessary to explain complex logic
- Keep documentation up-to-date with code changes

## Development Workflow

### Making Changes
1. Review existing code structure before making changes
2. Make minimal, focused changes that address the specific issue
3. Ensure changes don't break existing functionality
4. Test changes when applicable

### Git Practices
- Write clear, descriptive commit messages
- Keep commits focused on a single logical change
- Reference issue numbers in commit messages when applicable

## Testing

As the project grows, testing infrastructure should be established. When adding tests:
- Follow testing patterns established in the repository
- Ensure tests are focused and maintainable
- Tests should validate the specific functionality being added or changed
